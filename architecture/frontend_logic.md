# 프론트엔드 로직

## 헤더 검색과 키워드 필터

헤더 검색은 현재 선택 폴더와 무관하게 전체 폴더를 대상으로 실행한다. `keyword`, `semantic`, `rag` 호출 모두 선택된 `folderId`를 보내지 않는다.

상세 필터 UI는 키워드 검색에만 적용한다. 필터 메뉴 안에서 값을 변경해도 즉시 검색하지 않고, `적용`을 눌렀을 때만 저장된 필터 상태로 검색을 재실행한다.

키워드 필터 상태:

- `fileTypes`: `pdf`, `image`, `text`. 빈 배열이면 전체 파일 유형.
- `createdFrom`, `createdTo`: 등록일 기준 date input 값.
- `processingStatuses`: `ready`, `processing`, `failed`. 빈 배열이면 전체 상태.

처리 상태 기본값은 `ready`다. 필터 UI의 `전체`는 상태 조건을 적용하지 않는다는 의미이며, payload에서는 `processing_statuses: []`로 전달한다.

필터 메뉴 구성:

- `유형`: 전체, PDF, 이미지, 텍스트.
- `처리 상태`: 전체, 준비됨, 처리 중, 실패.
- `등록일`: 시작일, 종료일.
- `초기화`: 기본 필터 상태로 되돌린다.
- `적용`: 현재 query가 있고 검색 모드가 keyword이면 키워드 검색을 재실행한다.

`검색 지우기`는 query와 검색 결과만 지우며 키워드 필터는 유지한다. 필터 상태 초기화는 필터 메뉴의 `초기화`에서만 수행한다.

## 검색 결과 표시

검색 API는 청크 단위 결과를 반환할 수 있지만, 결과 그리드는 고유 문서 row 기준으로 렌더링한다. 검색 결과 개수 문구도 `searchResults.length`가 아니라 화면에 표시되는 고유 문서 row 수를 사용한다.

검색 결과 그리드에서만 상태 컬럼 우측에 `폴더` 컬럼을 표시한다. 일반 폴더 탐색 그리드에는 폴더명 컬럼을 표시하지 않는다. 루트 문서는 `내 드라이브`, 폴더 목록과 매칭되지 않는 문서는 `알 수 없음`으로 표시한다.

문서 목록과 문서 메타데이터 패널은 파일 수정일 대신 등록일을 표시한다.

## 폴더 생성 상태 갱신

- 폴더 생성 성공 후에는 생성된 폴더로 자동 진입하지 않고 현재 `selectedFolderId`, `documents`, `selectedDocumentId`를 유지한다.
- 새 폴더는 `api.createFolder` 응답을 `mergeFolderIntoList`로 `folders` 상태에만 반영해 사이드바와 현재 하위 폴더 목록에 즉시 보여준다.
- 폴더 생성은 문서 목록이나 검색/RAG 결과를 바꾸는 작업이 아니므로 현재 위치의 문서 재조회는 하지 않는다.
