# 프론트엔드 미구현 항목

현재 소스코드와 `architecture/` 문서 기준의 남은 작업입니다.

## 높음

### 처리 실패 문서 재처리

처리 실패 문서를 다시 처리하는 버튼과 상태 흐름이 없습니다. 백엔드 재처리 API도 현재 `409`를 반환합니다.

### 작업 상태 UX

업로드와 AI 생성은 `processing` 문서 반환 후 3초 폴링으로 완료를 확인합니다. 장기 작업의 단계별 진행률, 취소, 재시도 UX는 없습니다.

## 중간

### 검색 필터

현재 검색은 선택 폴더 중심이고 키워드/의미/RAG 모드를 지원합니다. 문서 유형, 태그, 날짜, 상태 필터 UI는 없습니다.

### 폴더 이동 UI

백엔드 API와 `lib/api.ts`는 `parent_id` 변경을 지원하지만, UI는 폴더 이름 변경만 제공합니다. 폴더를 다른 부모로 옮기는 조작이 없습니다.

### 청크/추출 텍스트 보기

백엔드는 `/documents/{document_id}/content`를 제공합니다. 프론트엔드는 검색 결과와 RAG 인용에서 일부 청크 내용을 보여주지만, 문서 상세에서 전체 추출 텍스트와 청크 목록을 보여주지는 않습니다.

### 계보 상세

생성 문서의 작업, 모델, 출처 문서, 프롬프트는 볼 수 있습니다. 제공자, 생성 파라미터, 워크플로 DNA, 출처 청크 ID/내용 상세는 표시하지 않습니다.

## 낮음

### 실패 복구 UX

처리 오류는 표시할 수 있지만 원인별 안내나 재시도 흐름은 없습니다.

### MinIO 오류 UX

보기/다운로드는 백엔드 URL을 사용합니다. presigned URL 만료나 MinIO 장애에 대한 별도 안내는 없습니다.
