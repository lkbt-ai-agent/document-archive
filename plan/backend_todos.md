# 백엔드 미구현 항목

현재 소스코드와 `architecture/` 문서 기준의 남은 작업입니다.

## 높음

### 저장 문서 재처리

`POST /api/v1/documents/{document_id}/process`는 존재하지만 `409`를 반환합니다. 저장된 원본을 다시 읽어 추출, 메타데이터 생성, 청킹, 임베딩을 재실행해야 합니다.

현재 오류 문구에는 "synchronously"가 남아 있어 `BackgroundTasks` 기반 업로드 처리와도 맞지 않습니다.

### 전용 작업 큐

업로드와 AI 생성은 FastAPI `BackgroundTasks`로 분리됐지만, 별도 큐/워커는 없습니다. 서버 프로세스 수명과 분리된 재시도, 진행률, 작업 취소, 장기 OCR/생성 안정성을 위해 큐가 필요합니다.

### 스캔 PDF OCR fallback

PDF는 `pypdf`만 사용합니다. 텍스트가 비어 있는 PDF를 이미지/OCR 경로로 넘기지 않습니다.

## 중간

### 검색 필터 확장

검색은 `folder_id`, `root_only`, `limit`만 받습니다. 문서 유형, 태그, 날짜, 상태 필터가 없습니다.

### 데이터베이스 인덱스

테이블 생성은 있지만 관계형 인덱스와 HNSW 벡터 인덱스가 정의되어 있지 않습니다.

### 스키마 마이그레이션

시작 시 `create_all()`과 일부 `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` 보정만 수행합니다. 운영 데이터 변경을 추적할 Alembic 마이그레이션이 필요합니다.

## 낮음

### 이미지 설명 제공자

이미지 OCR은 있지만 별도의 이미지 설명/분석 제공자는 없습니다.

### 생성 문서 처리 이력

생성 문서는 검색 가능하게 인덱싱되고 계보도 저장하지만, 단계별 처리 로그나 상태 전이 이력은 남기지 않습니다.

### 모듈 정리

일부 도메인 디렉터리는 비어 있습니다. 기능이 늘어나면 라우터, 서비스, 저장소 계층을 더 분리할 수 있습니다.
